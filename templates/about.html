About The Project:
{#
#}

A web portal to manage files and centralize analysis tasks related to immuno-repertoire sequencing projects in the lab



Primary Goals:

File Management - File Conversion, User Associations, File Metadata, Storage+Retrieval
Preprocessing - Quality Filter, Adapter Trimming, PEAR/Stitching Overlaps, (Kmer-based error correction?)
Annotating - alignment, V-gene calls, CDR calls, isotyping
Storing Data - Tracking raw and analyzed files, Automated backup
Postprocessing - crunching numbers, pairing VH:VL libraries, making mass spec (fasta uniq) database, ask questions on annotated files


Scope of Project:

File Management
Download from GSAF Link, Local Files
Job Output Files - Download Analysis Files
Track File Metadata
Pipeline Management
Preprocessing, Annotation, Analysis and DB Integration Pipelines (as far as a game plan goes, let's start small and work up)
Local asynchronous job execution


Keys to Project Success:

Centralize sequence data and analyses generated from BIGG-funded NGS
Standardize and make it easy to go from FASTQ to uniform annotation files and analysis summaries
Enable researchers to share data between each other on read-only and read-write levels
Login and see dashboard of tasks/pipelines/actions recently executed

Project Status Updates:

Initial Stack Considerations:


We are moving to a new computer nicknamed 'geordy' that will serve as the web app server, computation worker, mongo + postgres db servers, and primary file storage.
But we are mindful of the future and where we need to go from here:
Pipeline executions will be able to run on computing grid
Web server / file access not within our on-campus firewall / security guidelines
Jobs are asynchronous and can be scheduled without immediate resource dependencies


Initial Stack At Beginning Of Development:

Flask Web App
Celery / RabbitMQ Task Management
MongoDB


Stack Update June 2016:

Flask Web App Running On Scalable Heroku Server(s)
Cloud RabbitMQ Queue System, Can Route Tasks To Discreet Queues Based on UT or External User Status
AWS Compute Instance Configured And Imaged. Need to Install Celery / other dependencies on Geordy
Currently only linked to PostgreSQL server running on Heroku, but will soon have parsers for our mongo collections to build experiment, sequence and annotation records and stepwise migrate data to SQL (if we want).
